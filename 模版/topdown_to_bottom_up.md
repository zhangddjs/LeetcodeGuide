# 通用套路:机械转换四步

top-down 转 bottom-up 有一个几乎机械化的流程,任何记忆化递归都适用:

## 第一步:把递归函数的签名当作数组下标。

`compute(s1, s2, st) → dp[s1][s2][st]`

## 第二步:递归的基例 → 数组的初始化。

你的基例是 st == K 时返回 0 或 1,那就先把 `dp[·][·][K]` 这一整层填好。基例在递归里是"最先返回的",在迭代里就是"最先填的"。

## 第三步:分析依赖方向,决定循环顺序。

这是唯一需要动脑的一步。看转移式右边用到了哪些状态:

`compute(s1, s2, st)` 依赖 `compute(s1+1, ·, st+1)`、`compute(·, s2+1, st+1)`——三个维度都依赖更大的下标。

规则很简单:被依赖的必须先算。依赖更大的下标,循环就从大到小;依赖更小的,就从小到大。所以这题三重循环全部倒着走:st 从 K-1 到 0,s1 从 n 到 0,s2 从 m 到 0。

## 第四步:递归调用 → 数组读取,return → 数组写入。

return A + B 变成 `dp[s1][s2][st] = A + B`。入口 `compute(0,0,0)` 变成最后读 `dp[0][0][0]`。
